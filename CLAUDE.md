# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
# Run the Streamlit web app (primary entry point)
streamlit run app.py

# Run the CLI (secondary entry point)
python main.py screen --market US --roe 15 --gross-margin 30
python main.py analyze <TICKER>

# Install dependencies
pip install -r requirements.txt
# or with uv: uv sync

# Build / run via Docker
docker build -t value-investment .
docker run -p 8501:8501 value-investment
```

There is **no test suite** and **no lint config wired into CI**; `pytest`, `black`, and `ruff` are listed as optional dev deps in [pyproject.toml](pyproject.toml) but no tests exist.

## Authentication model

Every LLM call routes through [llm.py](llm.py) (`claude_complete` / `claude_complete_with_websearch`), which uses **`claude-agent-sdk`** with the local Claude Code CLI subscription — there is no `ANTHROPIC_API_KEY` path anywhere in production code. The Streamlit auth panel (`app.py` `main()`) only accepts a **subscription OAuth token** generated by `claude setup-token`; the token is written to `CLAUDE_CODE_OAUTH_TOKEN` on each rerun. Do not reintroduce `ANTHROPIC_API_KEY` handling.

Other API keys (Tavily, NewsAPI, Twelve Data, Morningstar, FMP) are managed in [admin.py](admin.py) — read with `get_api_keys()` and pushed into env via `apply_api_keys_to_env()` from `init_config()` on every Streamlit rerun.

## Architecture

### Entry points
- [app.py](app.py) — Streamlit web app, the primary UI. Contains a 3-step tabbed workflow (Screening → Anomaly Analysis → Summary Report), persistent AI chat at the bottom of every page, and an admin/settings panel.
- [main.py](main.py) — Click-based CLI for batch screening / single-ticker analysis. Independent of Streamlit; reuses the same domain modules.

### Data flow (3-step workflow)
1. **Screening** → [data_loader.py](data_loader.py) reads CSV/XLS from `data/` (default dir is `<repo>/data`, configurable via `DataLoader(data_dir=...)`). [screener.py](screener.py) applies the 10 criteria and uses [valuation.py](valuation.py) (in-house two-stage DCF: Low IV / High IV).
2. **Anomaly Analysis** → [anomaly_detector.py](anomaly_detector.py) runs rule-based checks; [ai_agents.py](ai_agents.py) `AnomalyAgent` runs Claude-powered forensic analysis on the 10-year history (XLS or FMP fallback).
3. **Summary Report** → [summary_report.py](summary_report.py) renders the Plotly/HTML dashboard; [peer_finder.py](peer_finder.py) + [chart_engine.py](chart_engine.py) drive competitor comparison; [enhanced_report.py](enhanced_report.py) generates the AI-enhanced DOCX report (uses `claude_complete` to fill the standardized 6-part template per company).

### LLM layer
- All Claude calls go through [llm.py](llm.py). Each call runs in its **own thread with a fresh asyncio event loop** so the SDK subprocess is fully torn down between calls. Built-in retry/backoff handles transient `Command failed with exit code 1` subprocess errors.
- `claude_complete` is text-only with `allowed_tools=[]` — empty allowlist forces a pure completion (otherwise the CLI burns the single turn on tool calls instead of answering).
- `claude_complete_with_websearch` enables `WebSearch` + `WebFetch` and defaults to `max_turns=30` for multi-step research (e.g. 10-year financials lookup). It is slow and burns subscription quota — use sparingly.
- Default model is `claude-sonnet-4-6` (`DEFAULT_MODEL`).

### Multi-agent system
[ai_agents.py](ai_agents.py) defines `ValueInvestmentAgent`, `ScreeningAgent`, `AnomalyAgent`, `ResearchAgent`. The app prefers `ResearchAgent` as the default chat agent because it has all capabilities without handoff issues. Agents declare local tools via `@tool` decorators that wrap calls into the same domain modules listed above.

### Optional integrations (graceful degradation)
- [fmp_api.py](fmp_api.py) — Financial Modeling Prep client; only initialized when `FMP_API_KEY` is set. Used for company profiles, 10-year financials fallback, and the "View Trends" popup.
- [enhanced_analysis.py](enhanced_analysis.py) — extended analysis utilities consumed by `app.py`.
- [report_generator.py](report_generator.py) — markdown/Word report helpers.

### Streamlit session conventions
- `st.session_state.workflow_data` carries `filtered_df`, `selected`, and `final` between the three steps.
- `st.session_state.persistent_chat_messages` is a rolling chat history; the bottom-of-page chat folds the **last 6 messages** into a single user prompt because `claude-agent-sdk`'s `query()` is one-shot per call (no native multi-turn memory).
- Workflow buttons read **"Proceed to ..."** (not "Move to ..."). Keep that wording.

## Data files

Bundled sample data lives under [data/](data/):
- `US All In One Screeners *.csv` and `SG All In One Screeners *.csv` — screener inputs (most recent file wins via glob sort).
- `Companies with anomalies*.xls` — 30-year financial history fed to the anomaly analyzer.

[.dockerignore](.dockerignore) excludes top-level `*.csv` / `*.xls` but explicitly re-includes `data/*.csv` / `data/*.xls` so the bundled data ships in the image. Don't undo that re-inclusion.
